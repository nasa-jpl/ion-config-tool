Describing Command Parameters

Most of the dictionary is dedicated to carefully describing each parameter for each configuration command. These settings are important for the tool, so that editting can be done more easily and safely.The following explains each column heading for describing parameters. One key tool idea is the use of so-called clone values. Often a parameter value references an identifier established elsewhere in the model, such as a node number or a port number. These special values are called clones, so that if the original identifier is later changed (e.g. a port number re-assigned), all of its clones are automaticaly changed as well. This is a key technique for maintaining the correctness of the complete configuration.

ParametersShort description of the parameter.
RequiredIs the parameter value required or optional.
ValueTypeIs value text, number, datetime or special. Supports editing control.
UnitsNumeric values can have units (secs, bytes, ...).
DefaultThe default value when the command is omitted.
CopyCloneThe parameter is automatically assigned a clone value. (e.g.node num).
PickCloneThe user selects the clone value from a model-based list.
